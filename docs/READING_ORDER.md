# Порядок изучения проекта (Гайд для студента)

Чтобы быстро разобраться в проекте и успешно защититься, изучай файлы в следующей последовательности:

## 0. ЛОКАЦИЯ И СИНХРОНИЗАЦИЯ (NEW)
Проект физически находится в `C:\Универ\SEM_6\schosi\PythonDSP`, но связан через Junction-ссылку со старым адресом. Изменения в любом из путей синхронны.

## 1. ТРЕБОВАНИЯ И ЗАДАНИЕ
*   `FULL_TASK_LIST.md` — Твой вариант (10) и список графиков.
*   `ALL_VARIANTS_DATA.md` — База данных всех вариантов для сравнения.

## 2. ГЛУБОКАЯ ТЕОРИЯ
*   `docs/MASTER_GUIDE.md` — Исчерпывающий разбор теории (Бабочка, Фурье, Фильтры). Читать ОБЯЗАТЕЛЬНО.

## 3. АРХИТЕКТУРА И ЛОГИРОВАНИЕ
*   `core/config_variants.py` — Как устроена база данных вариантов.
*   `main.py` — Главное меню и логика переключения окон.
*   `core/utils/aspects.py` — Система автоматического логирования (AOP).

## 4. МАТЕМАТИЧЕСКОЕ ЯДРО (АЛГОРИТМЫ)
*   `core/signals/generator.py` — Синтез сигналов.
*   `core/signals/fourier.py` — ДПФ и БПФ.
*   `core/signals/math_ops.py` — Свертка и корреляция.
*   `core/signals/filters.py` — Три типа фильтров.

## 5. ПОЛЬЗОВАТЕЛЬСКИЙ ИНТЕРФЕЙС
*   `labs/lab1_instruments.py` — Код первой лабораторной.
*   `labs/lab2_filters.py` — Код второй лабораторной.

## 6. ВЕРИФИКАЦИЯ (РЕЗУЛЬТАТЫ)
*   `results/lab1_report.txt` — Отчет по скорости и точности.
*   `results/lab2_report_var10.txt` — Отчет по SNR и коэффициентам.
*   `results/debug_logs/` — Текстовые дампы всех вычисленных массивов.
