# Глава 2. Архитектура программного комплекса

## 2.1 Слои системы
Проект организован по модульному принципу, что позволяет легко масштабировать его.

1.  **Core Layer (Ядро):** Математические алгоритмы (Фурье, фильтры, операции).
2.  **Lab Layer (Прикладной уровень):** Скрипты конкретных лабораторных работ с графическим интерфейсом.
3.  **Cross-cutting Layer (Аспекты):** Система прозрачного логирования данных.

## 2.2 Структура папок
*   `/core/signals/` — реализация DSP алгоритмов.
*   `/core/utils/` — инфраструктурные вещи (AOP логирование).
*   `/core/config_variants.py` — центральная база данных вариантов.
*   `/labs/` — исполняемые файлы с GUI.
*   `/results/` — автогенерируемые отчеты, аудио и графики.

## 2.3 Система логирования (AOP на Python)
Для отделения математики от побочных действий (записи на диск) использован подход, аналогичный Spring AOP в Java.
*   **Декоратор `@log_dsp_action`** — перехватывает выполнение функций.
*   **Context `DSPContext`** — хранит состояние (номер варианта, текущая лаба).
*   **Преимущество:** Мы можем включить или выключить логирование всего проекта, изменив один декоратор, не трогая формулы.
