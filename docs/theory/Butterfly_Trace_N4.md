# Пример работы алгоритма «Бабочка» (N = 4)

Этот пример соответствует схеме на **стр. 24** вашей методички. 

## 1. Входные данные
Пусть наш сигнал $x = [1, 2, 3, 4]$. 
Частота дискретизации $f_s = 4$ Гц.

## 2. Шаг 1: Разделение (Decimation)
Мы делим сигнал на четные и нечетные индексы:
*   $x_{even} = [x_0, x_2] = [1, 3]$
*   $x_{odd} = [x_1, x_3] = [2, 4]$

## 3. Шаг 2: Бабочки первого уровня (N = 2)
Для каждой пары считаем сумму и разность:
*   $E = [1+3, 1-3] = [4, -2]$
*   $O = [2+4, 2-4] = [6, -2]$

## 4. Шаг 3: Объединение (Бабочка N = 4)
Используем поворачивающий коэффициент $W_N^k = e^{-j\frac{2\pi}{N}k}$.
Для $N=4$: $W_4^0 = 1$, $W_4^1 = -j$.

**Верхняя половина (сумма):**
*   $X[0] = E[0] + W_4^0 \cdot O[0] = 4 + 1 \cdot 6 = \mathbf{10}$
*   $X[1] = E[1] + W_4^1 \cdot O[1] = -2 + (-j) \cdot (-2) = \mathbf{-2 + 2j}$

**Нижняя половина (разность):**
*   $X[2] = E[0] - W_4^0 \cdot O[0] = 4 - 1 \cdot 6 = \mathbf{-2}$
*   $X[3] = E[1] - W_4^1 \cdot O[1] = -2 - (-j) \cdot (-2) = \mathbf{-2 - 2j}$

## 5. Итог (Спектр)
$X = [10, -2+2j, -2, -2-2j]$
Если мы посчитаем это через ДПФ, результат будет **таким же**. «Бабочка» просто сгруппировала слагаемые так, чтобы не считать лишнего.

---

### Почему это работает для N = 32?
В случае с $N=32$ алгоритм сделает 5 таких уровней ($2^5 = 32$). Сложность упадет с 1024 операций до 160. Совпадение графиков 6 и 19 в вашей лабе доказывает, что эта цепочка «бабочек» не теряет точность.
